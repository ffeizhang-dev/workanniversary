# 📋 如何复制本地文件代码

## 方法1：使用记事本/文本编辑器（最简单）

### 步骤：

1. **找到文件**
   ```
   C:\Users\zhangfeifei\work-anniversary-system\assessment-survey.html
   ```

2. **右键点击文件** → 选择 **"打开方式"**

3. **选择编辑器**：
   - **记事本** (Notepad)
   - 或 **Notepad++**
   - 或 **VSCode**
   - 或 **任何文本编辑器**

4. **全选代码**
   - 按 `Ctrl + A` （全选）

5. **复制代码**
   - 按 `Ctrl + C` （复制）

6. **完成！** 现在代码已复制到剪贴板

## ⚠️ 注意事项

1. **确保复制完整**
   - 使用 `Ctrl + A` 全选，确保不漏掉任何内容
   - 从第一行 `<!DOCTYPE html>` 到最后一行 `</html>`

2. **不要用Word打开**
   - Word会改变代码格式
   - 必须使用纯文本编辑器（记事本、Notepad++、VSCode等）

3. **检查文件大小**
   - 原文件大小约 50-100 KB
   - 如果复制后内容很少，说明没复制完整

4. **粘贴到GitHub时**
   - 先删除GitHub编辑器中的旧内容
   - 然后粘贴新内容
   - 确保从头到尾都是新代码

## 🆘 常见问题

### Q1: 打不开文件怎么办？
**A**: 确保文件是 `.html` 格式，右键选择"打开方式" → "记事本"

### Q2: 复制后粘贴到GitHub显示乱码？
**A**: 使用记事本打开，确保文件编码是 UTF-8

### Q3: 记事本打开后看到很多代码，正常吗？
**A**: 正常！HTML文件就是这样，全选复制即可

### Q4: 可以直接拖拽文件到GitHub吗？
**A**: 可以！直接在GitHub页面点击 "Upload files"，拖拽文件上传更简单

## 🚀 更简单的方法：直接上传文件

4. GitHub会自动覆盖同名文件
5. 输入提交信息，点击 "Commit changes"

**这样更简单，不需要复制粘贴！**
