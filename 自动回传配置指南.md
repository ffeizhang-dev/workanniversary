# 📡 自动回传配置指南

## 🎯 功能概述

自动回传功能可以让评估人填写完问卷后，系统自动将数据发送给组织者，无需手动导出和发送JSON文件。

### 三种回传方式

1. **飞书机器人** - 推荐方式 ✅
   - 实时自动发送
   - 数据格式化展示
   - 安全可靠

2. **自定义Webhook** - 开发者友好
   - 灵活的数据接收
   - 适合技术团队

3. **邮件发送** - 备用方案
   - 需要手动操作
   - 兼容性好

---

## 🚀 方式一：飞书机器人（推荐）

### 第一步：创建飞书群机器人

1. **创建群聊**
   - 在飞书中创建一个评估数据收集群
   - 只需要HR/组织者在群中即可

2. **添加自定义机器人**
   ```
   群设置 → 群机器人 → 添加机器人 → 自定义机器人
   ```

3. **配置机器人**
   - 机器人名称：匿名评估数据收集器
   - 描述：自动接收评估问卷数据
   - 安全设置：关键词（输入"评估"）

4. **复制Webhook地址**
   ```
   https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

### 第二步：在系统中配置

1. **打开配置页面**
   - 在评估问卷页面点击"⚙️ 自动回传设置"按钮
   - 或直接访问 `auto-submit-config.html`

2. **选择"飞书机器人"模式**
   - 粘贴Webhook地址到输入框
   - 点击"🧪 测试连接"验证配置
   - 看到"✅ 连接测试成功"后点击"💾 保存配置"

### 第三步：分发问卷

将 `assessment-survey.html` 文件发送给评估人：

```
【匿名评估问卷 - 自动回传版】

请按照以下步骤完成评估：

1. 双击打开 assessment-survey.html 文件
2. 填写被评估人信息和评分
3. 点击"✅ 提交评估"按钮
4. 系统会自动将数据发送到飞书群

无需手动导出，数据会自动回传！
感谢您的参与！
```

每当有人提交评估，飞书群会自动收到消息卡片：


### 优势

✅ **实时接收** - 提交即可看到
✅ **格式美观** - 消息卡片排版清晰

✅ **可追溯** - 保留评估编号
✅ **完全匿名** - 只有评估内容，无提交者信息

## 🔧 方式二：自定义Webhook

### 适用场景

- 技术团队有自己的数据收集服务
- 需要与其他系统集成
- 需要自定义数据处理逻辑

### 配置步骤

1. **准备Webhook接收端点**
   - 创建一个HTTP接口接收POST请求
   - 接口URL示例：`https://your-domain.com/api/assessment/receive`

2. **数据格式说明**

系统会发送以下JSON数据：

```json
{
  "type": "anonymous_assessment",
  "data": {
    "assessmentId": "ANON-A3B9DF-1K2L3M",
    "employeeName": "张三",
    "department": "技术部",
    "position": "技术总监",
    "assessmentDate": "2025-02-25",
    "isAnonymous": true,
    "scores": {
      "dimension1": 4,
      "dimension2": 5,
      "dimension3": 4,
      "dimension4": 5,
      "dimension5": 4,
      "dimension6": 5
    },
    "totalScore": 27,
    "submittedAt": "2025-02-25T06:30:00.000Z"
  },
  "timestamp": "2025-02-25T06:30:00.000Z"
}
```

3. **服务端示例代码**

**Node.js (Express)**
```javascript
const express = require('express');
const app = express();

app.use(express.json());

app.post('/api/assessment/receive', (req, res) => {
  const { type, data, timestamp } = req.body;

  if (type === 'anonymous_assessment') {
    console.log('收到新评估:', data);

    // 保存到数据库
    // await saveToDatabase(data);

    // 发送通知
    // await sendNotification(data);

    res.json({ success: true, message: '数据已接收' });
  } else {
    res.status(400).json({ success: false, message: '未知数据类型' });
  }
});

app.listen(3000);
```

**Python (Flask)**
```python

from datetime import datetime

@app.route('/api/assessment/receive', methods=['POST'])
def receive_assessment():
    data = request.get_json()

    if data.get('type') == 'anonymous_assessment':
        assessment_data = data.get('data')
        print(f"收到新评估: {assessment_data}")

        # 保存到数据库
        # save_to_database(assessment_data)

        return jsonify({'success': True, 'message': '数据已接收'})

    return jsonify({'success': False, 'message': '未知数据类型'}), 400

if __name__ == '__main__':
    app.run(port=3000)
```

4. **在系统中配置**
   - 选择"自定义Webhook"模式
   - 输入Webhook URL
   - 测试连接
   - 保存配置

## 📧 方式三：邮件发送

### 工作原理

由于浏览器安全限制，无法直接发送邮件。此方式会打开用户的邮件客户端，自动填充内容。

### 配置步骤

1. **在配置页面选择"邮件发送"**
2. **填写接收邮箱**
   - 例如：hr@company.com
3. **自定义邮件主题**
   - 例如：新的匿名评估提交

### 使用体验

评估人提交后，会自动打开邮件客户端（如Outlook、Foxmail等），邮件内容已自动填写：

```
收件人: hr@company.com
主题: 新的匿名评估提交

被评估人: 张三
部门: 技术部
职位: 技术总监
评估编号: ANON-A3B9DF-1K2L3M

评分详情:
- 战略目标的承接者: 4 分
- 经营的责任者: 5 分
- 产品交付的责任者: 4 分
- 技术创新的引领者: 5 分
- 组织的领导者: 4 分
- 跨部门协作的统筹者: 5 分

总分: 27/30 分
提交时间: 2025-02-25 14:30:00
```

用户只需点击"发送"按钮。

### 局限性

⚠️ **需要手动操作** - 用户需要点击"发送"按钮
⚠️ **依赖邮件客户端** - 需要本地安装邮件软件
⚠️ **格式可能改变** - 不同邮件客户端格式不同

建议仅作为备用方案使用。

## 🔒 安全性说明

### 数据安全

✅ **完全匿名** - 不记录IP、设备、浏览器信息
✅ **无身份追踪** - 只有随机评估编号
✅ **本地优先** - 数据先保存在浏览器localStorage
✅ **传输加密** - HTTPS加密传输（如果使用）

### Webhook安全建议

如果使用自定义Webhook：

1. **使用HTTPS** - 确保传输加密
2. **验证签名** - 添加密钥验证请求来源
3. **限流保护** - 防止恶意大量提交
4. **数据验证** - 验证数据格式和范围

示例代码：

```javascript
const crypto = require('crypto');

function verifySignature(payload, signature, secret) {
  const hmac = crypto.createHmac('sha256', secret);
  hmac.update(JSON.stringify(payload));
  const computedSignature = hmac.digest('hex');
  return signature === computedSignature;
}

app.post('/api/assessment/receive', (req, res) => {
  const signature = req.headers['x-signature'];
  const secret = process.env.WEBHOOK_SECRET;

  if (!verifySignature(req.body, signature, secret)) {
    return res.status(401).json({ error: '签名验证失败' });
  }

  // 处理数据...
});
```

## 🐛 常见问题

### Q1: 飞书机器人没有收到消息？

**A**: 检查以下几点：
1. Webhook地址是否正确复制
2. 浏览器控制台是否有错误
3. 飞书机器人安全设置是否正确（关键词：评估）
4. 点击配置页面的"测试连接"按钮验证

### Q2: 自动回传失败怎么办？

**A**: 系统会自动降级为本地保存：
- 数据仍然保存在localStorage中
- 用户可以点击"💾 导出我的数据"手动导出
- 不会丢失任何数据

### Q3: 可以同时配置多种回传方式吗？

**A**: 当前版本只支持一种回传方式。建议使用飞书机器人，因为：
- 最稳定可靠
- 格式最美观
- 用户体验最好

### Q4: 如何测试配置是否成功？

**A**:
1. 在配置页面点击"🧪 测试连接"按钮
2. 飞书群会收到测试消息
3. 看到"✅ 连接测试成功"提示

### Q5: 评估人需要配置吗？

**A**: 不需要！只需要组织者配置一次：
- 组织者在配置页面设置好回传方式
- 将配置好的 `assessment-survey.html` 发送给评估人
- 评估人直接填写提交即可

配置会保存在HTML文件打开的浏览器localStorage中。

### Q6: 如何清除配置？

**A**:
- 方式1：在配置页面选择"不启用自动回传"并保存
- 方式2：浏览器按F12 → Console → 输入 `localStorage.removeItem('autoSubmitConfig')`

### Q7: Webhook超时怎么办？

**A**:
- 检查Webhook服务是否正常运行
- 确认URL可以从外网访问
- 查看服务端日志排查问题
- 临时禁用自动回传，使用手动导出

### Q8: 邮件方式没有打开邮件客户端？

**A**:
- 检查电脑是否安装邮件客户端（Outlook、Foxmail等）
- 浏览器可能阻止了打开邮件客户端
- 改用飞书机器人或Webhook方式

## 🎓 最佳实践

### 推荐配置方案

**小团队（<20人）**
- 使用飞书机器人
- 创建专门的数据收集群
- 实时查看提交情况

**中型团队（20-100人）**
- 使用飞书机器人 + 手动导出备份
- 定期使用收集工具合并导出Excel
- 做好数据备份

**大型团队（>100人）**
- 使用自定义Webhook
- 搭建专门的数据收集服务
- 自动存储到数据库
- 提供数据分析Dashboard

### 流程建议

1. **准备阶段**
   - 组织者配置好自动回传
   - 测试连接确保可用
   - 准备好评估人名单

2. **发放阶段**
   - 发送HTML文件给评估人
   - 附上清晰的填写说明
   - 告知截止时间

3. **收集阶段**
   - 实时监控飞书群消息
   - 统计已提交人数
   - 提醒未提交的评估人

4. **分析阶段**
   - 使用收集工具合并数据
   - 导出Excel进行分析
   - 生成评估报告
