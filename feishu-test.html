🔧 飞书回传测试工具

诊断和修复飞书机器人回传问题

⚠️ CORS跨域问题说明

如果看到"CORS policy"错误，这是浏览器安全限制，不是配置问题。

✅ 立即解决（选择一个）：

方法1：启动本地服务器（推荐）

双击 启动服务器.bat 文件，然后访问：

http://localhost:8000/feishu-test.html

方法2：使用curl命令测试（无CORS限制）

打开命令行，运行：

curl -X POST "你的Webhook" -H "Content-Type: application/json" -d "{\"msg_type\":\"text\",\"content\":{\"text\":\"测试评估\"}}"

方法3：部署到GitHub Pages

查看 在线部署指南.md，部署后就没有CORS问题了。

📝 输入飞书Webhook地址

💡 如何获取Webhook地址？

1. 在飞书群中点击右上角 "..." → 群设置
2. 找到"群机器人" → 添加机器人 → 自定义机器人
3. 设置名称和描述
4. 安全设置选择"关键词"，输入：评估
5. 复制生成的Webhook地址

💡 解决方案

问题1：Webhook URL格式错误

正确格式应该是：

https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

问题2：关键词验证失败

飞书机器人安全设置中必须添加关键词"评估"，步骤：

  1. 打开飞书群 → 群设置 → 群机器人
  2. 找到你的机器人 → 编辑
  3. 安全设置 → 关键词 → 输入：评估
  4. 保存设置

问题3：网络或CORS问题

如果浏览器控制台显示CORS错误：

  1. 确认是在本地文件（file://）打开，不是服务器（http://）
  2. 如果部署到服务器，需要后端代理转发请求
  3. 或者使用Chrome浏览器，右键 → 检查 → Console查看错误

问题4：机器人被禁用

检查机器人状态：

  1. 飞书群 → 群设置 → 群机器人
  2. 确认机器人显示为"已启用"
  3. 如果被禁用，点击启用

问题5：消息发送频率限制

飞书机器人有频率限制：

  1. 单个机器人：每分钟最多20条消息
  2. 如果超过限制，等待1分钟后重试
