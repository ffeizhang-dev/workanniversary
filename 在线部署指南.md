## 🚀 方案一：GitHub Pages部署（推荐）

### 优势
- ✅ 完全免费
- ✅ 稳定可靠（GitHub提供）
- ✅ HTTPS自动配置
- ✅ 支持自定义域名
- ✅ 无需信用卡

### 第一步：创建GitHub账号

如果还没有GitHub账号：
1. 访问 https://github.com
2. 点击 "Sign up" 注册账号
3. 验证邮箱

### 第二步：创建仓库

1. **登录GitHub后，点击右上角 "+"，选择 "New repository"**

2. **填写仓库信息**
   ```
   Repository name: work-anniversary-assessment
   Description: 员工能力匿名评估系统
   Public (选择公开)
   ✅ Add a README file (勾选)
   ```

3. **点击 "Create repository"**

### 第三步：上传文件

1. **在仓库页面，点击 "Add file" → "Upload files"**

2. **上传以下文件**（从本地文件夹拖拽上传）：
   - `assessment-survey.html`
   - `assessment-collect.html`
   - `assessment-results.html`
   - `auto-submit-config.html`
   - `多人评估使用指南.md`
   - `自动回传配置指南.md`

3. **在底部填写提交信息**
   ```
   Commit message: 上传评估系统文件
   ```

4. **点击 "Commit changes"**

### 第四步：启用GitHub Pages

1. **进入仓库设置**
   - 点击仓库页面的 "Settings" 标签

2. **找到 Pages 设置**
   - 在左侧菜单找到 "Pages"

3. **配置部署源**
   ```
   Source: Deploy from a branch
   Branch: main (或 master)
   Folder: / (root)
   ```

4. **点击 "Save"**

5. **等待1-2分钟**，刷新页面，会看到：
   ```
   Your site is live at https://your-username.github.io/work-anniversary-assessment/
   ```

### 第五步：访问链接

复制生成的URL，例如：
```
https://zhangfeifei.github.io/work-anniversary-assessment/assessment-survey.html
```

**完成！**现在可以将这个链接分享给评估人了。

### 配置自定义域名（可选）

如果有自己的域名（如 `assessment.yourcompany.com`）：

1. **在域名服务商（如阿里云、腾讯云）添加CNAME记录**
   ```
   类型: CNAME
   主机记录: assessment
   记录值: your-username.github.io
   ```

2. **在GitHub Pages设置中**
   - Custom domain: 填入 `assessment.yourcompany.com`
   - ✅ Enforce HTTPS（勾选）
   - 保存

3. **等待DNS生效**（10-30分钟）

## ⚡ 方案二：Vercel部署

### 优势
- ✅ 最快的全球CDN
- ✅ 自动HTTPS
- ✅ 每次更新自动部署
- ✅ 更好的性能

### 部署步骤

1. **访问 https://vercel.com**
   - 使用GitHub账号登录

2. **点击 "Add New..." → "Project"**

3. **导入GitHub仓库**
   - 选择刚才创建的 `work-anniversary-assessment` 仓库
   - 点击 "Import"

4. **配置项目**
   ```
   Framework Preset: Other
   Build Command: (留空)
   Output Directory: (留空)
   ```

5. **点击 "Deploy"**

6. **等待部署完成**（约30秒）

7. **获取链接**
   ```
   https://work-anniversary-assessment.vercel.app
   ```

### 绑定自定义域名

1. 在项目设置中点击 "Domains"
2. 输入你的域名
3. 按提示在域名服务商添加DNS记录
4. 完成

## 🔐 安全与隐私

### GitHub Pages安全性

✅ **完全安全**：
- HTTPS加密传输
- 代码开源可审计
- 数据存储在评估人本地浏览器
- GitHub不会收集评估内容

### 数据存储位置

- **localStorage**：存储在评估人浏览器本地
- **飞书消息**：发送到您的飞书群（如果启用）
- **GitHub Pages**：只托管HTML/CSS/JS代码，不存储评估数据

### 匿名性保证

- ✅ 不记录IP地址
- ✅ 不记录浏览器指纹
- ✅ 不记录访问时间（除非提交）
- ✅ 只有评估编号和内容

## 🐛 常见问题

### Q1: GitHub Pages部署后，页面404？

**A**: 等待1-2分钟让部署完成，然后：
- 确认文件名正确（区分大小写）
- 访问完整URL，如 `.../assessment-survey.html`
- 检查仓库是否设置为Public

### Q2: 自动回传不工作？

**A**: 在线版本需要重新配置：
- 每个评估人首次访问时需要配置
- 或使用URL参数传递配置（需要修改代码）
- 建议先手动测试配置是否正确

### Q3: 可以限制访问权限吗？

**A**: GitHub Pages是公开的，但可以：
- 使用难猜的URL（如添加随机字符）
- 使用Vercel/Netlify的密码保护功能
- 或使用Firebase Hosting的身份验证

### Q4: 如何更新已部署的文件？

**A**:
- **GitHub Pages**: 在仓库中上传新文件，自动更新
- **Vercel/Netlify**: Git push后自动重新部署
- 用户可能需要刷新浏览器缓存（Ctrl+F5）

### Q5: 评估人的数据会丢失吗？

**A**: 不会，因为：
- 数据存储在浏览器localStorage中
- 除非清除浏览器缓存，否则一直保留
- 建议评估人及时"导出我的数据"备份

### Q6: 可以统计有多少人访问了吗？

**A**:
- GitHub Pages不提供访问统计
- 可以集成Google Analytics（需要修改代码）
- 或使用Vercel/Netlify的内置分析功能

### Q7: 如何回收旧的评估链接？

**A**:
- 直接删除GitHub仓库（不推荐）
- 替换HTML文件内容为"评估已结束"页面
- 修改HTML添加截止日期检查逻辑
