# 飞书集成配置指南

## 📱 集成方式选择

系统支持两种飞书集成方式，根据您的需求选择：

### 方式一：飞书机器人 Webhook（推荐）

**适用场景**：
- ✅ 快速集成，5分钟完成配置
- ✅ 发送到指定群聊
- ✅ 无需开发权限
- ✅ 适合小团队使用

**限制**：
- ❌ 只能发送到群聊，不能私聊个人
- ❌ 消息发送频率有限制（20条/分钟）

### 方式二：飞书开放平台API

**适用场景**：
- ✅ 发送私聊消息给指定员工
- ✅ 更高的消息发送频率
- ✅ 支持更复杂的交互功能

**限制**：
- ❌ 需要企业管理员权限
- ❌ 需要创建企业自建应用
- ❌ 配置相对复杂

---

## 🚀 方式一：Webhook机器人配置（推荐）

### 步骤1：创建飞书群聊

1. 打开飞书APP或PC客户端
2. 创建一个新的群聊（例如：「周年祝福通知群」）
3. 将需要接收通知的HR同事拉入群聊

### 步骤2：添加群机器人

1. 进入群聊设置
2. 点击「群机器人」→「添加机器人」
3. 选择「自定义机器人」
4. 设置机器人名称：`入职周年提醒`
5. 上传机器人头像（可选）

### 步骤3：获取Webhook地址

添加机器人后，系统会生成一个Webhook地址，格式如下：

```
https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxx
```

**重要提示**：
- ⚠️ 请妥善保管此URL，不要公开分享
- ⚠️ 该URL相当于群机器人的密钥

### 步骤4：在系统中配置

1. 登录入职周年祝福系统
2. 进入「系统设置」
3. 找到「飞书集成」选项卡
4. 选择「Webhook机器人」模式
5. 粘贴Webhook URL
6. 点击「测试连接」验证
7. 点击「保存配置」

### 步骤5：测试发送

配置完成后，您可以：
- 点击「发送测试消息」按钮
- 检查飞书群聊是否收到测试消息
- 如果收到，说明配置成功！

---

## 🔧 方式二：开放平台API配置

### 步骤1：创建企业自建应用

1. 访问飞书开放平台：https://open.feishu.cn/
2. 登录企业管理员账号
3. 进入「开发者后台」→「企业自建应用」
4. 点击「创建企业自建应用」

### 步骤2：配置应用信息

**基本信息**：
- 应用名称：`入职周年祝福系统`
- 应用描述：`自动发送员工入职周年祝福消息`
- 应用图标：上传系统Logo

**权限配置**（重要）：

进入「权限管理」，申请以下权限：

| 权限 | 说明 | 必需 |
|-----|------|------|
| `im:message` | 获取与发送单聊、群组消息 | ✅ |
| `im:message:send_as_bot` | 以应用身份发消息 | ✅ |
| `contact:user.base:readonly` | 获取用户基本信息 | ✅ |
| `contact:user.employee_id:readonly` | 获取用户员工ID | ✅ |

### 步骤3：获取凭证

在「凭证与基础信息」页面获取：

- **App ID**（应用ID）：例如 `cli_a1b2c3d4e5f6g7h8`
- **App Secret**（应用密钥）：例如 `1a2b3c4d5e6f7g8h9i0j`

⚠️ **安全提示**：App Secret相当于密码，请勿泄露！

### 步骤4：发布应用

1. 点击「版本管理与发布」
2. 创建应用版本
3. 填写版本说明
4. 提交审核
5. 等待企业管理员审核通过

### 步骤5：在系统中配置

1. 登录入职周年祝福系统
2. 进入「系统设置」→「飞书集成」
3. 选择「开放平台API」模式
4. 填写：
   - App ID
   - App Secret
5. 点击「测试连接」
6. 点击「保存配置」

### 步骤6：获取员工User ID

为了给员工发送私聊消息，需要获取他们的飞书User ID：

**方法1：通过飞书管理后台**
1. 进入飞书管理后台
2. 「通讯录」→ 找到员工
3. 查看员工的「用户ID」

**方法2：通过API查询**
```javascript
// 系统会提供一个查询工具
// 输入员工邮箱或手机号，自动查询User ID
```

在系统的员工信息中，添加一个新字段「飞书User ID」，填入对应的ID。

## 💡 使用场景

### 场景1：群聊通知（Webhook）

**配置**：
- 模式：Webhook机器人
- Webhook URL：群机器人地址

**效果**：
- 每天自动检查当日周年的员工
- 在飞书群聊中发送祝福消息卡片
- 所有群成员都能看到

**适合**：HR团队内部通知

### 场景2：私聊祝福（API）

**配置**：
- 模式：开放平台API
- App ID + App Secret
- 员工信息中填写飞书User ID

**效果**：
- 系统直接私聊员工发送祝福
- 更加个性化和私密
- 员工收到专属祝福

**适合**：正式的员工关怀

### 场景3：混合模式

**配置**：
- 同时配置Webhook和API
- 在发送祝福时选择发送方式

**效果**：
- 可以选择发群聊或私聊
- 灵活应对不同场景

## 🔐 安全建议

### Webhook安全

1. **不要公开分享Webhook URL**
   - URL包含在配置中，只有系统管理员可见
   - 如果泄露，立即在飞书中重置机器人

2. **IP白名单**（可选）
   - 在飞书机器人设置中添加IP白名单
   - 只允许服务器IP发送消息

3. **签名验证**（高级）
   - 飞书支持消息签名验证
   - 系统可配置签名密钥进行双向验证

### API安全

1. **App Secret保护**
   - 存储在服务器环境变量中
   - 不要硬编码在前端代码
   - 定期更换密钥

2. **权限最小化**
   - 只申请必需的权限
   - 定期审查权限使用情况

3. **日志审计**
   - 记录所有消息发送日志
   - 包括时间、接收者、内容摘要

## 🛠️ 故障排查

### 问题1：测试连接失败

**可能原因**：
- Webhook URL错误或过期
- 网络连接问题
- 机器人被移除

**解决方法**：
1. 检查URL是否正确复制
2. 在飞书中确认机器人仍在群聊中
3. 尝试重新创建机器人

### 问题2：消息发送成功但群里看不到

**可能原因**：
- 消息格式错误
- 群机器人被禁言
- 飞书服务延迟

**解决方法**：
1. 检查消息JSON格式
2. 确认机器人没有被禁言
3. 等待几秒后刷新群聊

### 问题3：API返回403 Forbidden

**可能原因**：
- 权限未申请或未通过
- Access Token过期
- App ID/Secret错误

**解决方法**：
1. 检查应用权限配置
2. 验证凭证是否正确
3. 重新获取Access Token

### 问题4：消息发送频率限制

**限制说明**：
- Webhook：20条/分钟
- API：根据企业等级不同

**解决方法**：
1. 系统会自动限速（1秒/条）
2. 分批发送大量消息
3. 升级企业版获取更高配额

## 💬 高级功能

### 定时发送

**配置定时任务**：
1. 进入「系统设置」→「自动提醒」
2. 设置发送时间：例如每天上午9:00
3. 选择发送方式：飞书群聊或私聊

2. 创建新模板，选择「飞书卡片」类型
3. 使用Markdown语法编写内容
4. 支持变量：`{name}`, `{years}`, `{department}` 等

**添加按钮（高级）**：
- 「查看员工档案」按钮
- 「发送祝福」按钮

需要使用飞书开放平台API模式。
