# 🔧 飞书配置"Failed to fetch"错误解决方案

## ❌ 错误说明

**错误信息**：`连接测试失败: Failed to fetch`

**原因**：浏览器CORS跨域限制
- 从本地文件（file://）无法直接访问飞书API
- 这是浏览器安全机制，不是配置错误

**重要**：Webhook地址本身是正确的，只是浏览器环境限制了测试功能。

---

## ✅ 解决方案（3种方法）

### 方案1：使用curl命令测试（推荐，最简单）

#### Windows系统

1. **打开命令提示符**
   - 按 `Win + R`
   - 输入 `cmd`
   - 按回车

2. **复制并运行以下命令**（替换Webhook地址）：
   ```bash
   curl -X POST "你的Webhook地址" -H "Content-Type: application/json" -d "{\"msg_type\":\"text\",\"content\":{\"text\":\"入职周年提醒系统测试 - 连接成功！\"}}"
   ```

3. **如果返回**：
   ```json
   {"code":0}
   ```
   **说明连接成功！** ✅

**示例**：
```bash
curl -X POST "https://open.feishu.cn/open-apis/bot/v2/hook/xxxxx-xxxx-xxxx" -H "Content-Type: application/json" -d "{\"msg_type\":\"text\",\"content\":{\"text\":\"入职周年提醒系统测试 - 连接成功！\"}}"
```

## 🔍 故障排查

### 问题1：curl返回错误码

#### code: 9499
```json
{"code":9499,"msg":"Bad Request"}
```
**原因**：消息未包含关键词

**解决**：
- 检查飞书机器人安全设置中的关键词
- 确保消息包含该关键词
- 例如关键词是"周年"，消息必须包含"周年"

**正确命令**：
```bash
curl -X POST "你的Webhook" -H "Content-Type: application/json" -d "{\"msg_type\":\"text\",\"content\":{\"text\":\"入职周年提醒测试\"}}"
```

#### code: 19021
```json
{"code":19021,"msg":"Verification failed"}
```
**原因**：Webhook地址错误或已过期

**解决**：
1. 重新复制Webhook地址
2. 确认机器人还在群里
3. 如有疑问，删除机器人重新添加

---

### 问题2：飞书群收不到消息

**检查清单**：
- [ ] 机器人是否还在群里
- [ ] 机器人是否被禁言
- [ ] 消息是否包含关键词
- [ ] Webhook地址是否正确

**解决步骤**：
1. 进入飞书群设置
2. 群机器人 → 找到你的机器人
3. 确认状态显示"正常"
4. 查看安全设置中的关键词
5. 重新测试，确保消息包含关键词

## ⚙️ 配置文件说明

配置保存在浏览器localStorage中：

**存储位置**：
- 键名：`feishuConfig`
- 内容：JSON格式的配置信息

**手动配置**（高级）：

如果你熟悉浏览器开发者工具，可以：

1. 打开 `feishu-config.html`
2. 按 `F12` 打开开发者工具
3. 切换到"控制台"(Console)
4. 运行以下代码：

```javascript
// 保存Webhook配置
localStorage.setItem('feishuConfig', JSON.stringify({
  mode: 'webhook',
  webhookUrl: '你的Webhook地址',
  enabled: true
}));

alert('配置已保存！');
```

5. 刷新页面，配置生效
