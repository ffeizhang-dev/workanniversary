# 📊 员工能力匿名评估系统

一个完全匿名、支持多人在线填写、自动回传数据的员工能力评估问卷系统。

## ✨ 核心特性

- 🔒 **完全匿名** - 不记录任何身份信息，只生成随机评估编号
- 🌐 **在线访问** - 部署后通过URL直接访问，无需下载文件
- 📱 **自动回传** - 支持飞书机器人/Webhook/邮件自动发送数据
- 👥 **多人评估** - 支持分布式数据收集或集中云端汇总
- 📊 **可视化分析** - 自动生成统计图表和评级
- 💾 **灵活导出** - 支持CSV、Excel、JSON多种格式

## 🚀 快速开始

### 方式一：本地使用（无需部署）

1. **下载所有文件到本地文件夹**

2. **组织者配置自动回传**
   - 打开 `auto-submit-config.html`
   - 选择飞书机器人模式
   - 输入Webhook地址
   - 测试并保存

3. **分发给评估人**
   - 将 `assessment-survey.html` 文件发送给评估人
   - 评估人双击打开文件填写
   - 提交后自动发送到飞书群

### 方式二：在线部署（推荐）

#### 步骤1：部署到GitHub Pages

1. **创建GitHub仓库**
   ```
   仓库名: work-anniversary-assessment
   公开 (Public)
   ```

2. **上传所有HTML文件**
   - 拖拽上传到仓库

3. **启用GitHub Pages**
   ```
   Settings → Pages → Source: main branch
   ```

4. **获取访问链接**
   ```
   https://your-username.github.io/work-anniversary-assessment/
   ```

#### 步骤2：生成评估链接

1. **打开链接生成器**
   ```
   https://your-username.github.io/work-anniversary-assessment/url-generator.html
   ```

2. **配置回传方式**
   - 输入基础URL
   - 选择飞书机器人
   - 输入Webhook地址
   - 点击"生成链接"

3. **复制生成的链接**
   ```
   示例：
   https://your-username.github.io/work-anniversary-assessment/assessment-survey.html?autoSubmit=feishu&webhook=...
   ```

4. **分享给评估人**
   - 通过邮件/微信/飞书发送链接
   - 评估人点击即可填写
   - 提交后自动回传数据

## 📊 评估维度

系统包含6个评估维度，每个维度0-5分：

1. **战略目标的承接者** - 理解和执行公司战略的能力
2. **经营的责任者** - 对业务结果负责的意识
3. **产品交付的责任者** - 保证产品质量和交付
4. **技术创新的引领者** - 推动技术创新和改进
5. **组织的领导者** - 团队管理和领导力
6. **跨部门协作的统筹者** - 协调各部门资源

**评分标准**：
- 0分：从未展现
- 1分：偶尔展现
- 2分：经常展现
- 3分：表现优秀
- 4分：超出预期
- 5分：卓越非凡

**评级体系**：
- 85%+ (26-30分)：卓越非凡
- 70-84% (21-25分)：表现优秀
- 50-69% (15-20分)：符合预期
- <50% (<15分)：需要改进

## 🔧 使用场景

### 场景1：小团队评估（<20人）

**推荐方案**：飞书机器人自动回传

1. 创建飞书群机器人
2. 生成带配置的评估链接
3. 发送给评估人
4. 实时在飞书群查看评估数据

### 场景2：中型团队（20-100人）

**推荐方案**：在线链接 + 飞书回传 + 定期导出

1. 部署到GitHub Pages
2. 使用链接生成器生成URL
3. 评估人通过链接填写
4. 定期使用收集工具导出Excel汇总

### 场景3：大型团队（>100人）

**推荐方案**：自定义Webhook + 数据库存储

1. 搭建数据接收服务
2. 配置Webhook自动回传
3. 数据自动存储到数据库
4. 提供数据分析Dashboard

## 🔒 隐私与安全

### 匿名性保证

✅ **不记录**：
- IP地址
- 浏览器指纹
- 设备信息
- 评估者身份

✅ **只记录**：
- 随机评估编号（ANON-XXXXXX-YYYYYYY）
- 被评估人信息
- 评分内容
- 提交时间

### 数据存储

- **本地模式**：数据存储在评估人浏览器localStorage
- **在线模式**：数据通过飞书/Webhook发送到指定位置
- **GitHub Pages**：只托管静态文件，不存储评估数据

## 🐛 常见问题

### Q: 评估人需要注册账号吗？
A: 不需要，直接访问链接即可填写。

### Q: 如何查看有多少人已提交？
A:
- 飞书模式：查看飞书群消息数量
- 手动收集：统计收到的JSON文件数量
- Webhook模式：查询数据库记录数

### Q: 数据会丢失吗？
A: 不会，数据存储在浏览器localStorage中，除非清除浏览器缓存。建议评估人及时提交或导出。

### Q: 可以修改已提交的评估吗？
A: 提交后无法修改（保证数据一致性）。如需修改，清除浏览器localStorage后重新填写。

### Q: GitHub Pages访问速度慢怎么办？
A: 可以改用Vercel或Netlify部署，它们提供更快的全球CDN。

### Q: 如何防止恶意提交？
A:
- 使用难猜的URL（添加随机字符）
- 设置评估截止时间
- 在服务端验证数据合理性
- 使用Vercel/Netlify的访问限制功能
