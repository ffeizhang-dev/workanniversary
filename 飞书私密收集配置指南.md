# 🔒 飞书私密数据收集配置指南

## 问题说明

当前配置会将评估数据发送到公开的飞书群，所有成员都能看到评估内容，**不符合匿名评估的隐私要求**。

## ✅ 解决方案：创建专门的数据收集群

### 为什么这个方案最好？

- ✅ **最简单** - 5分钟完成配置
- ✅ **完全私密** - 只有HR/组织者看得到数据
- ✅ **实时收集** - 评估提交后立即收到
- ✅ **无需改代码** - 只需更换Webhook地址

---

## 📋 配置步骤（5分钟）

### 第1步：创建专门的数据收集群

1. **打开飞书** → 点击右上角 "+" → **创建群聊**

2. **群名称设置**：
   ```
   【私密】员工评估数据收集
   ```

3. **添加成员**：
   - ⚠️ **只添加HR和组织者**（例如：3-5人）
   - ❌ **不要添加评估人**
   - ❌ **不要添加被评估人**

4. **群设置**：
   - 开启"仅群主和管理员可邀请"
   - 建议设置为"私密群"（不可被搜索）

---

### 第2步：在数据收集群中添加机器人

1. **点击群聊右上角** "..." → **群设置**

2. **找到"群机器人"** → **添加机器人** → **自定义机器人**

3. **配置机器人**：
   ```
   名称：评估数据收集机器人
   描述：接收员工匿名评估数据，仅限HR查看
   ```

4. **安全设置**（重要！）：
   - 选择 **"关键词"**
   - 输入关键词：`评估`
   - ⚠️ 不要选"签名验证"（会导致发送失败）

5. **点击"添加"** → 复制生成的 **Webhook地址**

   示例：
   ```
   https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

---

### 第3步：生成新的评估链接

1. **打开链接生成器**：
   ```
   https://你的用户名.github.io/仓库名/url-generator.html
   ```

2. **配置参数**：
   - **基础URL**：你的GitHub Pages地址
   - **回传方式**：选择"飞书机器人"
   - **Webhook地址**：粘贴刚才复制的**新Webhook**（来自私密群）

3. **点击"生成链接"** → 复制生成的新链接

---

### 第4步：分享新链接给评估人

使用新链接替换之前的旧链接，发送给评估人：

```
【员工能力匿名评估】

各位同事好！

为了更全面地了解团队成员的能力表现，我们开展匿名评估活动。

📝 评估链接：
https://你的用户名.github.io/仓库名/assessment-survey.html?autoSubmit=feishu&webhook=...

✅ 完全匿名 - 不记录任何身份信息
🔒 数据私密 - 仅HR可见，不会公开
⏱️ 5分钟完成 - 6个维度，每个0-5分

截止时间：2024年XX月XX日

谢谢配合！
```

## ⚠️ 常见问题

### Q1: 旧链接已经发出去了怎么办？

**方案A（推荐）**：通知评估人使用新链接
```
【重要通知】
之前的评估链接请勿使用，请使用以下新链接：
[新链接]

原因：为了更好地保护隐私，数据收集方式有调整。
已填写的同事请忽略此消息。
```

**方案B**：同时保留两个链接
- 旧链接数据会发到公开群（但会继续收集）
- 新链接数据发到私密群
- 评估结束后合并两个来源的数据

### Q2: 私密群的机器人会被评估人发现吗？

不会。机器人只在添加它的群里工作，评估人：
- 看不到这个私密群
- 看不到机器人
- 看不到任何提交结果

### Q3: 可以中途更换Webhook吗？

可以，但需要：
1. 生成新的评估链接（使用新Webhook）
2. 通知评估人使用新链接
3. 已提交的数据无法迁移（需手动复制或接受分散在两个群）

### Q4: 如果评估人比较多（>50人），会不会刷屏？

不会影响你的正常工作：
- 数据只在私密收集群显示
- 可以关闭该群的消息通知
- 评估结束后统一查看

建议：
- 评估期间：关闭群通知，每天定时查看
- 或者：设置"仅@消息提醒"

## 🔐 安全检查清单

在开始正式评估前，请确认：

- [ ] 私密数据收集群只有HR/组织者（3-5人）
- [ ] 已在私密群中添加机器人并获取新Webhook
- [ ] 使用url-generator.html生成了新链接
- [ ] 新链接包含新的Webhook地址
- [ ] 已用新链接测试，数据发送到私密群
- [ ] 确认旧的公开群不再收到数据
- [ ] 已将新链接发送给评估人

## 📞 技术支持

如果遇到问题：

1. **测试Webhook是否有效**：
   ```bash
   curl -X POST "你的新Webhook" -H "Content-Type: application/json" -d '{"msg_type":"text","content":{"text":"测试 评估"}}'
   ```

   应该返回：`{"code":0}`

2. **检查群机器人状态**：
   - 飞书群 → 群设置 → 群机器人
   - 确认机器人显示"已启用"

3. **查看浏览器控制台**：
   - F12 → Console
   - 查看是否有错误信息
