## 🎯 数据共享

三个页面共享同一个 localStorage 数据：

### 数据结构

```javascript
// 员工数据（由员工管理页面维护）
localStorage.getItem('employees')
// 格式：JSON数组
// [
//   {
//     id: "uuid",
//     name: "张三",
//     department: "技术部",
//     position: "工程师",
//     joinDate: "2021-01-15",
//     reminderEnabled: true,
//     ...
//   }
// ]

// 飞书配置（由飞书配置页面维护）
localStorage.getItem('feishuConfig')
// 格式：JSON对象
// {
//   enabled: true,
//   mode: "webhook",
//   webhookUrl: "https://open.feishu.cn/open-apis/bot/v2/hook/xxx",
//   ...
// }
```

### 数据流转

```
员工管理 → 添加员工 → 保存到localStorage
                            ↓
飞书配置 → 配置Webhook → 保存到localStorage
                            ↓
祝福确认 → 读取员工数据 → 读取飞书配置 → 发送祝福
```

## ⚠️ 注意事项

### 1. 浏览器缓存问题

如果部署后访问页面还是旧版本：

**解决方法**：
- 强制刷新：`Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac)
- 清除缓存：浏览器设置 → 清除缓存和Cookie
- 使用无痕模式：`Ctrl + Shift + N` (Chrome) 或 `Ctrl + Shift + P` (Firefox)

### 2. localStorage 是基于域名的

- ✅ **正确**：所有页面都通过 `ffeizhang-dev.github.io` 访问，数据共享
- ❌ **错误**：混合使用本地文件（`file://`）和在线版本（`https://`），数据不共享

**建议**：始终使用在线版本，不要本地打开文件。

### 3. 导航链接是相对路径

导航链接使用的是文件名，例如：
```html
<a href="work-anniversary-reminder.html">员工管理</a>
```

这意味着：
- ✅ 如果所有文件在同一目录（仓库根目录），可以正常跳转
- ❌ 如果文件在不同目录，需要修改为完整路径

### 4. 部署时间

- GitHub Pages 部署通常需要 **2-3 分钟**
- 可以在仓库的 "Actions" 标签查看部署状态
- 绿色✅表示部署成功，红色❌表示失败

## 🆘 常见问题

### Q1: 导航栏没有显示？

**A**: 检查：
1. 文件是否正确上传（包含完整的HTML代码）
2. 浏览器是否缓存了旧版本（强制刷新）
3. CSS样式是否正确加载（F12打开开发者工具查看）

### Q2: 点击导航链接跳转404？

**A**: 检查：
1. 三个HTML文件是否都已上传到GitHub
2. 文件名是否正确（区分大小写）
3. 文件是否在同一目录

### Q3: 数据不同步？

**A**: 确保：
1. 所有页面都通过同一个域名访问（都是`ffeizhang-dev.github.io`）
2. 没有混合使用本地文件和在线版本
3. 浏览器允许使用localStorage（检查浏览器设置）

### Q4: 移动端导航显示不正常？

**A**:
1. 确认CSS中包含了移动端适配代码（`@media`查询）
2. 清除浏览器缓存
3. 尝试横屏查看
