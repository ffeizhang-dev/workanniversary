# 🎯 飞书@人功能使用指南

## ✅ 功能说明

现在系统支持在发送周年祝福时**自动@对应员工**，让祝福消息更加醒目！

---

### 1. 员工管理页面
- ✅ 添加了"飞书User ID"字段
- ✅ 支持保存和编辑飞书ID
- ✅ 提供了获取方法的提示链接

### 2. 祝福确认页面
- ✅ 发送祝福时自动检测员工是否配置了飞书ID

- ✅ 消息卡片完整显示祝福内容

## 📝 如何获取飞书User ID

### 方法1：通过飞书管理后台（推荐）

1. **登录飞书管理后台**
   ```
   https://feishu.cn/admin
   ```

2. **进入通讯录**
   - 点击左侧菜单"通讯录"
   - 或直接访问：https://feishu.cn/admin/contacts

3. **找到员工**
   - 在通讯录中搜索员工姓名
   - 点击员工名片

4. **复制User ID**
   - 在员工详情页面，找到"User ID"字段
   - 格式：`ou_xxxxxxxxxxxx`（以ou_开头）
   - 点击复制

5. **粘贴到系统**
   - 在员工管理页面，编辑对应员工
   - 将User ID粘贴到"飞书User ID"字段
   - 保存

### 方法2：通过飞书开放平台

1. **访问开放平台**
   ```
   https://open.feishu.cn/
   ```

2. **使用API Explorer**
   - 进入"工具与资源" → "API Explorer"
   - 搜索接口：`/open-api/contact/v3/users/batch_get_id`

3. **调用接口查询**
   ```json
   {
     "emails": ["zhangsan@company.com"],
     "mobiles": []
   }
   ```

4. **获取返回的user_id**

### 方法3：通过飞书客户端

1. **打开飞书客户端**
2. **找到员工资料卡**
   - 搜索员工
   - 点击头像查看资料卡
3. **点击"更多信息"**
4. **查看User ID**（部分企业版本支持）

```

    ↓
```

   - 点击"批量发送全部"

## ⚙️ 飞书User ID 格式

### 正确格式

```
ou_1234567890abcdefghijklmnopqrstuv
```

**特点**：
- ✅ 以 `ou_` 开头
- ✅ 后面跟一串字母和数字
- ✅ 长度约30-40个字符

### 错误格式

```
❌ 12345678 （纯数字）
❌ user_id （字母）
❌ @张三 （飞书昵称）
❌ zhangsan@company.com （邮箱）
```

## 💡 常见问题

### Q1: 必须填写飞书User ID吗？

**A**: 不是必须的！
- ✅ 填写了：发送祝福时会@对应员工
- ✅ 不填写：正常发送祝福，只是不会@人
- 建议：为常用员工配置，提升祝福送达率

### Q2: 如何知道哪些员工配置了飞书ID？

**A**:
- 在员工管理页面，编辑员工时可以看到
- 或者在员工列表中添加显示标记
- 建议批量配置所有员工

### Q3: 飞书ID填错了会怎样？

**A**:
- 如果ID格式错误或不存在
- 消息会正常发送，但@不会生效
- 被@的人不会收到通知
- 不会影响其他功能

### Q4: @人需要特殊权限吗？

**A**:
- 飞书Webhook机器人默认支持@人
- 无需额外申请权限
- 只要员工在同一个飞书群即可

### Q5: 能@多个人吗？

**A**:
- 当前系统每个祝福只@对应员工
- 如果需要@多人，可以修改代码
- 或者手动在飞书群中再@其他人

### Q6: 能@所有人吗？

**A**:
- 不建议在祝福消息中@所有人
- 会造成通知打扰
- 建议只@对应员工

### Q7: 如何批量获取所有员工的飞书ID？

**A**: 可以通过飞书开放平台API批量获取：

```bash
# 使用飞书API批量查询
curl -X POST \
  https://open.feishu.cn/open-api/contact/v3/users/batch_get_id \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "emails": [
      "employee1@company.com",
      "employee2@company.com",
      "employee3@company.com"
    ]
  }'
```

返回结果：
```json
{
  "code": 0,
  "data": {
    "user_list": [
      {
        "email": "employee1@company.com",
        "user_id": "ou_xxxxxxxxxxxx"
      }
    ]
  }
}
```

### Q8: 飞书ID会变化吗？

**A**:
- 飞书User ID是固定的，不会变化
- 除非员工离职后重新入职
- 一次配置，长期有效

## 🔧 技术说明

### @人实现原理

飞书支持在消息中使用特殊标签来@人：

```javascript
// 在lark_md格式的消息中
const message = `<at user_id="ou_xxxxxxxxxxxx"></at>\n\n祝福内容...`;
```

### 消息发送逻辑

```javascript
// 1. 检查员工是否有飞书ID
if (employee.feishuUserId) {
    // 2. 在消息开头添加@标签
    messageContent = `<at user_id="${employee.feishuUserId}"></at>\n\n` + messageContent;
}

// 3. 发送到飞书
await sendToFeishu(messageCard);
```

### 数据结构

```javascript
// 员工数据（localStorage）
{
    id: "1234567890",
    name: "张三",
    department: "技术部",
    position: "高级工程师",
    feishuUserId: "ou_xxxxxxxxxxxx",  // ← 新增字段
    joinDate: "2021-02-26",
    reminderEnabled: true
}
```
